**submitRequest action** — signature required by useActionState (prevState first):

```typescript
export async function submitRequest(
  prevState: FormState,
  formData: FormData
): Promise<FormState>
```

Implementation steps (in order — DO NOT reorder):

1. **Extract fields** from formData:
   - `teacher_name`, `teacher_email`, `start_date`, `end_date` as strings
   - `leave_type` as `LeaveType` (cast from string)
   - `is_blackout`: compare `formData.get('is_blackout') === 'true'` — all FormData values are strings, never coerce with Boolean()
   - `reason`: `(formData.get('reason') as string) || null` — nullable

2. **Server-side validation** — collect all errors before returning:
   - `teacher_name`: required (trim check)
   - `teacher_email`: required (trim check)
   - `start_date`: required; if present, compare `new Date(start_date + 'T00:00:00')` against today with `today.setHours(0, 0, 0, 0)` — append T00:00:00 to avoid UTC off-by-one
   - `end_date`: required; if both start and end present, check `new Date(end_date + 'T00:00:00') < new Date(start_date + 'T00:00:00')`
   - `leave_type`: required
   - `is_blackout` unselected: `formData.get('is_blackout') === null` means teacher did not pick either radio — add error
   - If any errors, return `{ errors, values: { teacher_name, teacher_email, start_date, end_date, leave_type: leave_type ?? '', reason: reason ?? '' } }`

3. **Determine status**: `const status: RequestStatus = is_blackout ? 'auto_denied' : 'pending'`

4. **Insert into Supabase** (inside try/catch):
   ```typescript
   const supabase = createClient()
   const { error: dbError } = await supabase.from('requests').insert({
     teacher_name, teacher_email, start_date, end_date,
     leave_type, is_blackout, reason, status,
   })
   if (dbError) return { message: 'Something went wrong. Please try again.' }
   ```

5. **Send auto-denial email** (inside the same try block, AFTER the insert succeeds):
   - Only when `is_blackout === true`
   - `await sendEmail({ to: teacher_email, subject: 'Your time-off request — blackout period', html: autoDenialTemplate({ teacherName: teacher_name, leaveType: leave_type, startDate: start_date, endDate: end_date }) })`

6. **Redirect** — MUST be outside the try/catch block:
   ```typescript
   // Capture outcome before try/catch so redirect can use it after
   let outcome: 'pending' | 'auto_denied' = 'pending'
   try {
     // ... insert + email ...
     outcome = status
   } catch {
     return { message: 'Something went wrong. Please try again.' }
   }
   redirect(`/confirmation?status=${outcome}`)
   ```
   CRITICAL: redirect() throws NEXT_REDIRECT internally. If placed inside try/catch, the catch block swallows it and the redirect silently fails.
