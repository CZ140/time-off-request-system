const supabase = createClient()
const windowStart = new Date(Date.now() - 60_000).toISOString()
const { data: duplicate } = await supabase

  .select('id')
  .eq('teacher_email', teacher_email)
  .eq('start_date', start_date)
  .eq('end_date', end_date)

  .maybeSingle()

if (duplicate) {

}

Replace the current data-fetching block (lines 10-18) with the following try/catch pattern:

```typescript
const supabase = createClient()
let requests: RequestRow[] = []
let blackoutDates: BlackoutDateRow[] = []
let fetchError = false

try {
  const [{ data: requestsRaw, error: reqErr }, { data: blackoutDatesRaw, error: bdErr }] =
    await Promise.all([
      supabase.from('requests').select('*').order('submitted_at', { ascending: false }),
      supabase.from('blackout_dates').select('*').order('start_date', { ascending: true }),
    ])
  if (reqErr || bdErr) throw new Error('db')
  requests = (requestsRaw ?? []) as RequestRow[]
  blackoutDates = (blackoutDatesRaw ?? []) as BlackoutDateRow[]
} catch {
  fetchError = true
}
```

Then in the JSX return, add a conditional error message immediately before the `<TabSwitcher>` component inside `<main>`:

```tsx
{fetchError && (
  <p className="mb-4 text-sm text-red-600">Unable to load data. Please refresh.</p>
)}
```

The `<TabSwitcher>` should remain in place — it will render with empty arrays when fetchError is true, which is acceptable (it already handles empty arrays with "No requests found.").

Critical rules:
- Supabase returns errors as values (`{ data, error }`), NOT thrown exceptions. The `if (reqErr || bdErr) throw new Error('db')` line is required to convert Supabase error values into thrown exceptions so the catch block fires.
- Do NOT use a React Error Boundary — this is an async Server Component; try/catch in the component function is the correct pattern.
- The `type RequestRow` and `type BlackoutDateRow` type aliases at the top of the file must remain unchanged.
