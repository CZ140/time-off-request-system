**Test 1 — Admin notification email (REQ-03, EMAIL-02):**

**Test 2 — Approve flow (APPR-01, APPR-03, APPR-04, EMAIL-03):**
1. Click the Approve button in an admin notification email
2. Verify: browser redirects to /reviewed showing status "Approved", teacher name, dates, leave type, and admin email

4. Check the teacher's inbox: should receive the approval confirmation email ("Your time-off request has been approved")

**Test 3 — Idempotency (APPR-02):**
1. Click the same Approve link again (or the Deny link for the same already-approved request)
2. Verify: browser shows /reviewed page (same or another admin's email)
3. Verify: no second email in teacher's inbox
4. Verify: DB row status has NOT changed from its current value

**Test 4 — Deny flow (APPR-01, APPR-03, APPR-04, EMAIL-04):**
1. Submit another non-blackout request to get a fresh admin email
2. Click the Deny button
3. Verify: browser redirects to /reviewed showing status "Denied"
4. Check teacher inbox: denial email should include their dates and leave type

**Test 5 — Invalid token (APPR-01):**
1. In your browser, visit: http://localhost:3000/api/approve?action=approve&id=fake-id&token=wrongtoken&admin=test@test.com
